# 玄象所 命理整合網站

一個整合星座、塔羅、八字、姓名學、紫微斗數的命理原型網站，前端輸入資料後會呼叫 AI 生成個人化解讀。

## 這個資料夾要部署到 GitHub + Vercel（推薦方式）

```
MILI-V13/
├── index.html          ← 前端網頁（星座/塔羅/八字/姓名學/紫微斗數）
├── vendor/
│   └── iztro.min.js    ← 紫微斗數排盤引擎（開源套件，內建在專案裡，不依賴外部 CDN）
├── api/
│   └── interpret.js    ← Vercel 無伺服器函式，負責安全呼叫 Google Gemini API（金鑰放這裡，不會外流）
├── package.json         ← 標記為 ES Module，Vercel 部署 api/interpret.js 需要用到
└── README.md            ← 就是你現在看的這份
```

**部署前記得自己建立這兩個檔案**（Vercel 環境變數才是金鑰真正該放的地方，不要把金鑰寫進任何檔案裡）：
- `.gitignore`：內容寫 `.env` 跟 `.env.local` 兩行就好，避免不小心把本機測試用的金鑰檔案傳上 GitHub
- `.env.local`（僅本機測試用，正式環境用 Vercel 的 Environment Variables）：內容為 `GEMINI_API_KEY=你的金鑰`

## AI 解讀用的是哪個服務？（V2 更新）
V2 開始改用 **Google Gemini API**（`gemini-2.5-flash` 模型），原因是它有免費額度、不用綁信用卡，適合這種小流量的原型網站。

- 免費額度大約每分鐘 10-15 次請求、每天數百到 1500 次不等（Google 會不定期調整，正式上線前建議到 Google AI Studio 確認當下實際額度）
- 免費額度的輸入內容 Google 可能會用於改善模型，如果之後有隱私顧慮，可以在 Google Cloud 專案開啟計費來關閉這項條款
- 如果流量常常撞到限額，把 `api/interpret.js` 裡的 `MODEL` 常數改成 `"gemini-2.5-flash-lite"` 即可拿到更高的免費額度（品質略陽春）
- 金鑰申請：https://aistudio.google.com/apikey


## 版本紀錄
- **V13（目前版本）**：解讀內文（`.reading`，所有分頁生成的 AI 解讀文字都是這個樣式）字重從細體（300）改成正常體（400）。原本文字顏色其實已經是最深的墨色、對比度數值上不低，但因為中文字筆畫較多，用細體字顯示大段落文字時容易看起來「發虛」、長時間閱讀比較費眼力，改成正常字重後筆畫更扎實、閱讀起來更省力，不用再調整顏色或加大字級。
- V12：解決紫微命盤在小螢幕上「字太小看不清楚」的問題——① 命盤格子的最小尺寸與所有文字都調大一級（主星名稱、宮位名稱從 13px 調到 14.5px，其餘輔星/雜曜/大限文字也都放大）。② 新增「點擊放大」功能：點/點選任一宮位，會跳出一個置中的大視窗，用更大的字級（主星 19px）完整重新排版顯示該宮位的主星、輔星煞星、雜曜、大限、小限，點視窗外側或按 Esc 都能關閉，手機上不用再瞇眼看小字。③ 命盤下方新增操作提示文字，告知使用者可以左右滑動＋點擊放大。
- V11：紫微斗數全面改版——① 排盤引擎改用開源的 [iztro](https://github.com/SylarLong/iztro)（MIT License，已將 `iztro.min.js` 內建在 `vendor/` 資料夾，不依賴外部 CDN），輸入改成跟八字一致的「國曆出生日期＋時／分＋性別」，取代原本只能填農曆月份與時辰的陽春版本。② 新增完整十二宮命盤圖表：十四主星＋輔星（左輔右弼、文昌文曲、天魁天鉞）＋煞星（擎羊陀羅、火星鈴星、地空地劫）＋雜曜全部排出，命宮／身宮會特別標示並加上邊框強調，四化（祿權科忌）以彩色圓形標記在對應星曜旁，每個宮位也會顯示大限歲數範圍與小限。③ AI 解讀改成讀取命宮／財帛／官祿／夫妻／疾厄／福德六個關鍵宮位的實際主星與四化資料，並會自動算出使用者目前的虛歲、對應到哪個大限一併納入提示詞，解讀內容比舊版（只給命宮地支）更準確、更個人化。④ 命盤圖表較寬，手機上會自動變成左右滑動瀏覽，不會被擠壓變形。
- V10：星座運勢生成解讀時，會在結果欄上方畫出對應星座的原創 SVG 占星符號（例如牡羊座的雙角、雙魚座的雙魚等），外圍搭配雙圈刻度環（各自反向緩慢旋轉）與四個角落的星芒閃爍動畫，符號本身則以縮放＋淡入方式揭曉，呈現更有星象儀式感；同時將星座／八字出生日期欄位的預設範例從舊版寫死的日期改成通用示範日期 `19990101`，避免誤植成特定真實生日。
- V9：星座與八字的「出生日期」欄位改成可直接輸入 8 碼數字（例如 `19990101`），輸入時會自動補上斜線分隔，取代原本必須點開日曆選擇的方式，操作更直覺；同時會檢查日期是否真實存在（例如不會誤收 2 月 30 日）。八字的「出生時間」改用「時／分」兩個下拉選單取代原生時間選擇器，跨瀏覽器、跨裝置的顯示與操作更一致。
- V8：① 八字「五行分布圖」從長條圖改成正五邊形雷達圖——木火土金水依「相生」順序排在五邊形的五個頂點，資料多邊形會用 `requestAnimationFrame` 從中心展開到實際比例，背景另外畫出對應「相剋」順序的虛線星形參考線，視覺上更貼近命理上「五行」的概念，也更精簡地把五個數值的比例關係一次呈現出來。② 塔羅新增兩項功能：(a) 22 張大阿爾卡納牌翻牌後會顯示一枚原創的簡化 SVG 線條插圖（例如太陽、月亮、命運之輪、高塔等），逆位時插圖會整個旋轉 180 度，正位／逆位的顏色也不同，讓抽到的牌更好辨識；(b) 三張牌翻開後，會先用一排「重點特寫卡」把過去／現在／未來三張牌的圖示＋牌名＋關鍵字整理出來，AI 完整解讀文字生成後，還會額外抓出最後一句（通常是具體建議）獨立做成醒目的「重點提示」區塊，跟下方完整內文分開，方便使用者快速抓住重點而不必先讀完整段文字。
- V7：① 八字「五行分布圖」改成真正的 JS 動畫（`requestAnimationFrame` 逐格計算），長條寬度與右側數字會一起從 0 跑到目標值，並在跑分過程中加上光澤掃過效果，不再依賴容易被瀏覽器合併重繪、導致動畫失效的 CSS transition 技巧。② 姓名學結果新增「大名字招牌」與五格圖：姓名會用大字放大顯示在結果欄最上方，下方則有一個十字佈局的五格圖（天格／人格／地格／外格／總格），五個數理會以連線＋逐一彈出的動畫依序呈現，視覺上更清楚也更有儀式感。③ 星座與紫微斗數在結果生成時，新增一個由上而下擴散的「炫光閃現」動畫，並將等待文字換成帶有旋轉符文圖示的讀取提示，讓結果揭曉更有戲劇效果；此光暈與跑出動畫（chip 逐一淡入）也套用到其餘分頁的結構化標籤上，整站風格更一致。
- V6：① 修正八字面板「五行分布圖」長條偶爾不會跑動畫的問題——原本用兩層 `requestAnimationFrame` 觸發 CSS transition，在部分手機瀏覽器上會被瀏覽器合併成同一次重繪而不播放動畫，改成先強制觸發一次 reflow（讀取 `offsetWidth`）再套用目標寬度，動畫更穩定可靠。② 修正姓名學「手動輸入筆畫沒有用」的重大 bug——原本按下「計算五格並生成解讀」時，程式碼會無條件重新產生筆畫輸入欄位，把使用者剛剛手動填寫的筆畫值直接洗掉再讀取（永遠讀到空值或字典預設值），現在改成只有姓名字元真的改變時才重建欄位，手動輸入的筆畫會被正確保留並拿去計算。③ 筆畫字典新增一批常見名字用字（珍、維、冠、儀、雯…等），減少「未收錄請手動輸入」的情況。

## 已知限制（之後可以再優化）
- 八字：以國曆月份近似節氣月、日柱以公式估算，未逐一核對萬年曆
- 姓名學：筆畫字典只收錄常見字，生僻字會跳出手動輸入欄位讓使用者自己修正
- 塔羅牌的 SVG 插圖為簡化原創線條圖示，並非完整的塔羅牌美術風格，之後可以再加強成更精緻的插畫
- 紫微斗數：星曜亮度、四化與大限採 iztro 預設流派設定（較接近三合派），飛星派等其他流派算法可能略有出入；命盤本身的排演算法已是完整正規排法，這點跟八字／姓名學的簡化演算法性質不同

## 部署到 GitHub + Vercel 的完整步驟

### 1. 建立 GitHub Repository
到 https://github.com/new 建立一個新的 repository（可以設為 Private，只有你自己看得到程式碼）。

### 2. 把這個資料夾推上 GitHub

**電腦操作（推薦，最快）：**
```bash
cd MILI-V13
git init
git add .
git commit -m "V12：紫微命盤放大檢視 + 字級調整"
git branch -M main
git remote add origin https://github.com/你的帳號/你的repo名稱.git
git push -u origin main
```
> 不熟指令的話，下載「GitHub Desktop」（https://desktop.github.com/），用拖拉介面把整個資料夾發布上去，效果一樣。

**只有手機的話：**
1. 用手機瀏覽器（不是 GitHub App）打開你的 repo 網頁
2. `index.html`：點 **Add file → Upload files**，從手機裡選這個檔案上傳
3. `vendor/iztro.min.js`：用 **Add file → Create new file**，檔名欄位直接打 `vendor/iztro.min.js`（帶斜線，GitHub 會自動建立 `vendor` 資料夾），這支檔案內容很長，建議還是用電腦或 GitHub Desktop 上傳比較不容易複製貼上出錯
4. `api/interpret.js`：同樣用 **Add file → Create new file**，檔名打 `api/interpret.js`
5. `package.json`：同上，用 Create new file 建立
6. `.gitignore`：用 Create new file 建立，內容貼上兩行 `.env` 和 `.env.local`

**重要：`.env`／`.env.local` 檔案要被 `.gitignore` 排除，金鑰不會被上傳，這是故意的、也是必須的——金鑰只能存在 Vercel 的環境變數設定裡，絕對不要把金鑰貼進程式碼或上傳到 GitHub。**

### 3. 連接 Vercel
1. 到 https://vercel.com 用 GitHub 帳號登入
2. 點「Add New Project」→ 選擇你剛剛推上去的 repository
3. Framework Preset 選 "Other"（純 HTML，不是 React/Next.js）
4. 展開 "Environment Variables"，新增一筆：
   - Key: `GEMINI_API_KEY`
   - Value: 你的金鑰（AIza開頭那一串，申請網址在最下面）
5. 點 Deploy

幾十秒後 Vercel 會給你一個網址，例如 `https://mingli-website.vercel.app`，打開就能直接用，AI 解讀功能也會正常運作（因為 `/api/interpret` 跟前端在同一個網域下，不用另外設定網址）。

### 4. 之後要更新網站怎麼做？
以後只要修改檔案、`git push` 上去（或用手機重新上傳/編輯該檔案），Vercel 會自動偵測並重新部署，不用手動操作。

### 5. 想換成自己的網域？
Vercel 專案設定裡有「Domains」，把你的網域（例如 mingli.elvis-mis.com）填進去，照指示到你網域註冊商那邊加一筆 DNS 紀錄即可，Vercel 會自動處理 HTTPS 憑證。

## 疑難排解

**部署後點按鈕沒反應/一直顯示「生成中」？**
- 打開瀏覽器開發者工具（電腦版）看 Console 有沒有錯誤訊息
- 確認 Vercel 專案的 Environment Variables 裡有正確設定 `GEMINI_API_KEY`
- 確認金鑰本身有效、免費額度沒有當下被撞滿（到 https://aistudio.google.com 檢查，或看 Vercel 的 Function Logs 有沒有錯誤訊息）

**手機瀏覽器打開網址列顯示 content://download 而不是網址？**
- 這代表你打開的是「下載到手機的本機檔案」，不是真正上線的網站。等部署到 Vercel 後，要用 Vercel 給你的網址（https://xxx.vercel.app）打開才是正式運作的版本，本機檔案沒辦法呼叫後端 API。
